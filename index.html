üîß Control

You control how your pages are rendered; you can use any view framework (React, Vue, ...) and any tool (Vuex/Redux, GraphQL, Service Workers, ...).

Integrating tools is simple and natural.

Render Control

With vite-plugin-ssr, you define how your pages are rendered.

// _default.page.server.js
// Environment: Node.js

import { html } from 'vite-plugin-ssr'

// This could be any view framework: React 16, React 17, Vue 2, Vue 3, Svelte, Solid, Preact, ...
import { renderToHtml } from 'some-view-framework'

export async function render(pageContext) {
  const pageHtml = await renderToHtml(pageContext.Page)
  return html`<!DOCTYPE html>
    <html>
      <body>
        <div id="page-root">${html.dangerouslySkipEscape(appHtml)}</div>
      </body>
    </html>`
}
// _default.page.client.js
// Environment: Browser

import { hydrateToDom } from 'some-view-framework'

// This file defines the *entire* browser-side code.
// If we define an empty `_default.page.client.js` then our pages have zero browser-side code.

hydrate()

async function hydrate() {
  const pageContext = await getPage()
  const { Page } = pageContext
  await hydrateToDom(Page)
}

Natural Tool Integration

This control means we can easily integrate any tool we want.

Browser libraries

// _default.page.client.js
// Environment: Browser

// This is the first browser-side line of code.
// This is a good place to initialize error tracking such as Sentry or Bugsnag.
Sentry.init()
// And also for initializing a Service Worker.
navigator.serviceWorker.register('/sw.js')

hydrate()

// After hydration we can initialize vanilla JS component libraries, for example tooltips
tooltip.init(document.querySelectorAll('.tooltip')

hydrate() {
  // Here we can integrate performance measurement tools, e.g. to measure hydration performance
  // ...
}

This control over the browser-side allows us to integrate tools simply by following the tool's official installation guide; vite-plugin-ssr does not get in our way. In other words, integrating tools is natural.

No more hassle adding our favorite CSS framework or favorite browser library/technology.

Framework libraries

We have full control over how we setup and integrate our view framework (React/Vue/Svelte/...) which means we can use any framework tool (Apollo GraphQL, Vuex, Redux, PullState, Recoil, ...) simply by following their official SSR guide.

Server libraries

From a server perspective, vite-plugin-ssr is simply a server middleware.

const { createPageRender } = require('vite-plugin-ssr')

// On the server-side, `vite-plugin-ssr` is simply a `renderPage()` function.
const renderPage = createPageRender()

// `app` can be any server framework such as Express.js/Fastify/Koa/Hapi/...
app.get('*', async (req, res) => {
  // We can also use `renderPage()` in a serverless environment such as Cloudflare Workers
  const result = await renderPage({ url: req.url })
  res.send(result.renderResult)
})

Not only can we use vite-plugin-ssr with any server environment we want (Cloudflare Workers, EC2 instance, AWS lambda, Firebase, Express.js, Fastify, Hapi, ...), but we can also use any server tool we want, for example any authentication strategy & tool (email/password, OAuth, Passport.js, Grant, Auth0, ...).

SSR architecture that scales from small hobby projects to large-scale enterprise projects with highly custom precise SSR needs.

DX Scalability with Lazy-transpiling

Vite does lazy-transpiling: only code requested by the browser is transpiled. This means that if you define 10 pages but load only one page in the browser, then only the code that is needed to render that page is transpiled.

Thanks to Vite's lazy-transpiling approach, you can scale to (very!) large source code while with a dev speed and HMR that stays fast.

Scalable SSR Architecture

As you scale you may need increasinlgy precise SSR needs; vite-plugin-ssr is designed to be flexible for even highly custom SSR needs.

You can even, at very large scale, progressively replace vite-plugin-ssr with Vite's native SSR API which is a low level and robust (many projects, such as vite-plugin-ssr, and many companies are built on top of and depend on it). If you are Netflix and squeezing out each UX imperfection leads to a substantial revenue increase, then vite-plugin-ssr and/or Vite's SSR API are what you are looking for.
