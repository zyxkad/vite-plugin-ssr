From an architectural point of view, a vite-plugin-ssr app is simply a Node.js application.

In production, a vite-plugin-ssr app is simply two server middlewares: one middleware that serves the static files living at dist/client/, and a second middleware that server-side renders our pages.

This means we can deploy vite-plugin-ssr apps anywhere we want.

Some platforms, such as Cloudflare Workers or Vercel, bundle all server code into a single file, which means that all dependencies need to be statically determined at build-time. To that end, we can import dist/server/importer.js which loads all dependencies in a statically analysable way. Example:
